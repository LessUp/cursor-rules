# Rule Philosophy

Cursor Rules treats reusable `.mdc` files as governed engineering guidance. The repository is intentionally narrow in scope so the rules stay reviewable, composable, and maintainable over time.

## Core principles

1. Root-level rules are the product.
2. Rules should be reviewable like code.
3. Adoption paths matter more than raw rule count.
