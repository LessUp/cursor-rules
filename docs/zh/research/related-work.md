# 相关工作

Cursor Rules 借鉴了提示词仓库、Copilot 指令文件、技能系统和文档站工程化的优点，但它的目标更接近“规则策展库”而不是“提示词合集”。

## 可对照的几类实践

### Awesome Prompt / Prompt Collections

优点是入口轻、复制快；缺点是缺少工程结构、分类治理和长期维护边界。Cursor Rules 选择牺牲一点“即贴即用”的轻量感，换取更高的可审视性。

### 仓库级 AI 指令文件

如 `AGENTS.md`、`CLAUDE.md`、`GEMINI.md` 一类文件，它们非常适合表达仓库级行为约束，但不擅长做跨项目、跨语言的规则复用。Cursor Rules 的价值在于把这类经验进一步抽象成可复制规则。

### 工具内置技能 / 工作流

技能系统擅长表达“如何执行任务”；规则库擅长表达“在什么边界内执行任务”。二者结合，才能形成更稳定的 AI 协作体验。

## Cursor Rules 的差异化

1. 把根目录 `.mdc` 作为产品契约，而不是附属资产。
2. 把 Pages 站点做成导读、学院和白皮书，而不是单纯目录列表。
3. 让生成脚本和规则目录始终由同一事实源派生，避免说明文档与规则分叉。
