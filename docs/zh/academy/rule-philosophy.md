# 规则哲学

Cursor Rules 采用的是**策展式规则库**而不是“万能提示词列表”。这意味着它的目标不是尽可能多，而是尽可能清晰、稳定和可组合。

## 三条核心原则

### 1. 根目录规则文件就是产品

根目录 `.mdc` 文件必须保持公开、稳定、可复制。任何站点、脚本或说明文档，都只能围绕这层产物服务，而不能反过来成为真实来源。

### 2. 规则必须可审查

每条规则都应有清晰的 frontmatter、适用范围和正文结构。这样团队才能像 code review 一样 review 规则，而不是把 AI 协作约束变成黑盒。

### 3. 采用路径比规则数量更重要

多数团队失败，不是因为规则不够，而是因为没有阅读顺序和治理节奏。首页与学院页优先解释“为什么、何时、怎么用”，目录页才负责最后的选择动作。

## 为什么不追求“大而全”

大而全的 prompt 集合通常会带来三个问题：

1. 读者不知道该先用哪一条。
2. 重复、冲突和漂移会随着规模增长而积累。
3. 团队很难把局部经验沉淀为长期可维护契约。

Cursor Rules 选择更克制的模型：少量高质量规则 + 清晰分类 + 生成式目录 + 明确治理文档。
